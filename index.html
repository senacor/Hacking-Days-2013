Dieses Mal werden wir ein Spiel programmieren. Multi-Player, online und mit Anbindung in eine Datenbank. Als Client-Technologie stehen HTML5, CSS, JavaScript oder Java FX 2 zur Verfügung. Auf dem Server verwenden wir node.js (JavaScript) bzw. vert.x (Java, JavaScript, Groovy ... polyglott eben). Zwischen Client und Server werden wir mit WebSockets kommunizieren und Accounts und Highscores landen in einer NoSQL Datenbank (Riak). Da will noch jemand Responsive Web Design mit reinbringen? Ok ... merkt unsere HTML5 Client Anwendung, dass sie auf einem Smartphone läuft, wird kein Spielfeld, sondern ein Gamecontroller dargestellt!
